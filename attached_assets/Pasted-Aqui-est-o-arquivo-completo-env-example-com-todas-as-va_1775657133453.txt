Aqui está o arquivo completo .env.example com todas as variáveis necessárias para deixar seu SignalMaster Pro seguro.
1. Arquivo: .env.example
Crie este arquivo na raiz do seu backend (artifacts/api-server/.env.example):
env# =============================================
# SIGNALMASTER PRO v7 - ARQUIVO .env.example
# =============================================

# ==================== BANCO DE DADOS ====================
# PostgreSQL (recomendado) ou SQLite para desenvolvimento
DATABASE_URL="postgresql://usuario:senha@localhost:5432/signalmaster?schema=public"

# Para SQLite (mais simples para começar)
# DATABASE_URL="file:./dev.db"

# ==================== SEGURANÇA - JWT ====================
JWT_SECRET=suachavejwt_muito_longa_e_segura_aqui_2026
REFRESH_SECRET=outrachave_diferente_e_mais_longa_para_refresh_2026

# Tempo de expiração (não mude aqui, só no código se precisar)
JWT_EXPIRES_IN=15m
REFRESH_EXPIRES_IN=7d

# ==================== API KEYS (nunca exponha no frontend!) ====================
RESEND_API_KEY=re_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

# ==================== CONFIGURAÇÃO DO SERVIDOR ====================
NODE_ENV=development
PORT=8080

# URL do frontend (usado no CORS)
FRONTEND_URL=http://localhost:18264

# ==================== OUTRAS CONFIGURAÇÕES ====================
# Chave para criptografia adicional (opcional)
ENCRYPTION_KEY=outra_chave_super_secreta_32_caracteres_ou_mais

# ==================== AVISOS IMPORTANTES ====================
# 1. NUNCA commite este arquivo com valores reais no Git
# 2. Crie um arquivo .env baseado neste exemplo
# 3. Use senhas longas e aleatórias para JWT_SECRET e REFRESH_SECRET
# 4. Em produção, use variáveis de ambiente do servidor (Railway, Render, etc.)

2. Instruções Completa (Passo a Passo)
Passo 1: Criar o arquivo .env

Na pasta artifacts/api-server/, crie um arquivo chamado .env (sem a palavra "example")
Copie todo o conteúdo acima e cole no .env
Substitua os valores por informações reais:
